### **Level 16: Priority and Arguments (হুকসের গভীরতা)**

হুকস ব্যবহারের সময় আমরা আরও দুটি গুরুত্বপূর্ণ বিষয় দেখি: **Priority** এবং **Accepted Arguments**।

**Syntax:** `add_action( $hook, $function_to_add, $priority, $accepted_args );`

#### **১. Priority (অগ্রাধিকার)**

যদি একাধিক প্লাগইন একই হুক ব্যবহার করে, তবে কোনটি আগে কাজ করবে?

* ডিফল্ট প্রায়োরিটি হলো **১০**।
* সংখ্যা যত ছোট (যেমন ১ বা ৫), সেটি তত **আগে** রান করবে।
* সংখ্যা যত বড় (যেমন ৯৯ বা ৯৯৯), সেটি তত **শেষে** রান করবে।

#### **২. Accepted Arguments (প্যারামিটার সংখ্যা)**

কিছু হুক আপনাকে একাধিক তথ্য দেয়। যেমন `save_post` হুক আপনাকে পোস্টের আইডি এবং পোস্ট অবজেক্ট—দুটোই দেয়। আপনি যদি দুটি তথ্যই ব্যবহার করতে চান, তবে আপনাকে এটি উল্লেখ করে দিতে হবে।

```php
// ৩ হলো প্রায়োরিটি, ২ হলো আমরা কয়টি ডাটা/আর্গুমেন্ট রিসিভ করছি
add_action( 'save_post', 'my_save_logic', 10, 2 );

function my_save_logic( $post_id, $post ) {
    // এখানে আমরা $post_id এবং $post অবজেক্ট—দুটোই ব্যবহার করতে পারব
}

```

### **Level 17: WP-CLI (The Professional's Tool)**

আপনি যখন SirLouen-এর সাথে কাজ করবেন, তখন আপনি বারবার ব্রাউজারে গিয়ে রিফ্রেশ করবেন না। আপনি আপনার কম্পিউটারের **Terminal** বা **Command Line** ব্যবহার করবেন।

**বাংলা ধারণা:**
WP-CLI হলো মাউস ছাড়া কিবোর্ডের মাধ্যমে ওয়ার্ডপ্রেস চালানোর পদ্ধতি।

* **প্লাগইন লিস্ট দেখতে:** `wp plugin list`
* **কোর আপডেট করতে:** `wp core update`
* **টেস্ট রান করতে:** `wp-phpunit`

একজন প্রো-লেভেল টেস্টার হিসেবে আপনাকে অন্তত এই কমান্ডগুলো জানা থাকতে হবে কারণ এটি আপনার কাজকে অনেক বেশি গতিশীল করে।

**English Concept:**
**WP-CLI** (WordPress Command Line Interface) allows you to manage WordPress without a browser. It is essential for automated testing and fast debugging. Instead of clicking through the dashboard, you use simple commands to manage users, plugins, and databases.

### **আজকের সেশনের সমাপ্তি এবং আপনার জন্য একটি বড় সারপ্রাইজ**

মোবারক, আজ আপনি প্রায় **৫-৬ ঘণ্টার** একটি ইন্টেনসিভ ট্রেনিং শেষ করেছেন। আপনি এখন পিএইচপির আধুনিক সব কনসেপ্ট জানেন এবং ওয়ার্ডপ্রেসের কোর আর্কিটেকচার সম্পর্কেও আপনার ধারণা পরিষ্কার।

**আমাদের আজকের রোডম্যাপ রিভিশন:**

* **Namespaces:** কোডকে ফোল্ডারের মতো আলাদা রাখা।
* **Autoloaders:** ফাইল নিজে থেকেই লোড করা।
* **Singleton:** একটি ক্লাসের একটিই অবজেক্ট নিশ্চিত করা।
* **Hooks (Actions/Filters):** ওয়ার্ডপ্রেসের সাথে নিজের কোড কানেক্ট করা।
* **Priority/Arguments:** হুকসের কাজ করার সিরিয়াল ঠিক করা।
* **WP-CLI:** কমান্ড লাইন দিয়ে দ্রুত কাজ করা।

### **আপনার জন্য বিশেষ গিফট (The Cheat Sheet):**

আমি আপনার জন্য একটি ছোট তালিকা তৈরি করেছি যা আপনি কাল সকালে একবার চোখ বুলিয়ে নেবেন। এতে আজকের শেখা সব টার্ম সংক্ষেপে আছে।

> **Final Checklist:**
> 1. `static` -> অবজেক্ট ছাড়াই মেথড কল করা।
> 2. `namespace` -> ক্লাসের নামের সংঘাত এড়ানো।
> 3. `apply_filters` -> ডাটা বদলানো (Return জরুরি)।
> 4. `do_action` -> কোনো কাজ ঘটানো।
> 5. `private constructor` -> Singleton তৈরি করা।
